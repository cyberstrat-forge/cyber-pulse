**版本**: 1.0
**日期**: 2026-03-26
**状态**: 待审核
**阶段**: 阶段 1 - CLI 废弃 + 部署脚本调整

## 1. 概述

### 1.1 背景

cyber-pulse 经过多次功能迭代，需要检查和优化应用部署架构。本文档是三阶段优化计划的第一阶段。

### 1.2 目标

1. **明确部署与应用管理的边界**：部署通过脚本简化操作，应用管理通过 API 进行，完全废弃 CLI
2. **简化管理员 Key 机制**：单一真相来源，首次部署生成，忘记可重置
3. **分离开发者和运维人员体验**：开发者 clone 完整代码，运维人员仅获取部署管理包

### 1.3 范围

| 内容 | 阶段 1 |
|------|--------|
| CLI 废弃 | ✅ |
| 管理员 Key 机制重构 | ✅ |
| 新增 `api.sh` 管理脚本 | ✅ |
| 用户分离（安装脚本） | ✅ |
| 验证脚本调整 | ❌ 阶段 2 |
| 测试环境清理机制 | ❌ 阶段 2 |
| 升级流程审计 | ❌ 阶段 3 |

## 2. CLI 废弃

### 2.1 当前状态

CLI 模块位于 `src/cyberpulse/cli/`，包含以下命令：

| 命令 | 功能 | 替代方案 |
|------|------|---------|
| `source` | 情报源管理 | `api.sh sources` |
| `job` | 任务管理 | `api.sh jobs` |
| `client` | 客户端管理 | `api.sh clients` |
| `content` | 内容查询 | `api.sh items` |
| `log` | 日志管理 | `api.sh logs` |
| `diagnose` | 系统诊断 | `api.sh diagnose` |
| `config` | 配置管理 | 废弃 |

### 2.2 变更内容

| 操作 | 文件/模块 |
|------|----------|
| 删除 | `src/cyberpulse/cli/` 整个目录 |
| 移除入口点 | `pyproject.toml` 中的 `[project.scripts]` 条目 |
| 移除依赖 | typer, rich, prompt-toolkit |

### 2.3 镜像影响

| 方面 | 变化 |
|------|------|
| 镜像体积 | 减小（无 CLI 依赖） |
| 构建时间 | 减少 |
| CI/CD 工作流 | 无需修改 |

## 3. 管理员 API Key 机制

### 3.1 设计原则

| 原则 | 说明 |
|------|------|
| 单一真相来源 | 仅数据库存储（bcrypt 哈希） |
| Key 不可找回 | 创建时显示一次，后续无法查看 |
| 重置代替轮换 | 忘记 Key 时重置，生成新 Key |

### 3.2 存储方式

| 方式 | 是否使用 | 说明 |
|------|---------|------|
| `.env` 文件 | ❌ 移除 | 不再存储 `ADMIN_API_KEY` |
| 数据库 | ✅ | bcrypt 哈希存储 |

### 3.3 生命周期

```
┌─────────────────────────────────────────────────────────────────┐
│  首次部署                                                        │
├─────────────────────────────────────────────────────────────────┤
│  ./scripts/cyber-pulse.sh deploy                                │
│       ↓                                                         │
│  API 启动                                                        │
│       ↓                                                         │
│  ensure_admin_client() 检测无 admin client                      │
│       ↓                                                         │
│  生成 Key → bcrypt 哈希存入数据库 → 输出到终端一次                 │
│       ↓                                                         │
│  ╔═══════════════════════════════════════════════════════════╗ │
│  ║  Admin API Key: cp_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx  ║ │
│  ║  请立即保存，此 Key 仅显示一次                              ║ │
│  ╚═══════════════════════════════════════════════════════════╝ │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  忘记 Key / Key 泄露                                             │
├─────────────────────────────────────────────────────────────────┤
│  ./scripts/cyber-pulse.sh admin reset                           │
│       ↓                                                         │
│  提示确认：此操作将使旧 Key 失效                                  │
│       ↓                                                         │
│  确认后：生成新 Key → 更新数据库 → 输出一次                       │
│       ↓                                                         │
│  ╔═══════════════════════════════════════════════════════════╗ │
│  ║  新 Admin API Key: cp_live_yyyyyyyyyyyyyyyyyyyyyyyyyyyyyy ║ │
│  ║  请立即保存，此 Key 仅显示一次                              ║ │
│  ║  旧 Key 已失效                                             ║ │
│  ╚═══════════════════════════════════════════════════════════╝ │
└─────────────────────────────────────────────────────────────────┘
```

### 3.4 部署脚本 admin 命令

| 命令 | 说明 | 变化 |
|------|------|------|
| `admin show-key` | 显示当前 Key | ❌ 移除 |
| `admin rotate-key` | 轮换 Key | ❌ 移除 |
| `admin reset` | 重置 Key | ✅ 新增 |

### 3.5 代码变更

| 文件 | 变更 |
|------|------|
| `deploy/init/generate-env.sh` | 移除 `ADMIN_API_KEY` 生成 |
| `src/cyberpulse/api/startup.py` | 修改 `ensure_admin_client()`，生成 Key 并输出 |
| `scripts/cyber-pulse.sh` | 移除 `show-key`、`rotate-key`，新增 `reset` |

## 4. `api.sh` 管理脚本

### 4.1 技术选型

| 方案 | 选择 | 说明 |
|------|------|------|
| 实现 | 纯 Bash | 无需 Python 环境 |
| HTTP 客户端 | curl | macOS/Linux 自带 |
| JSON 处理 | jq | 需安装 |

### 4.2 配置文件

**位置**: `~/.config/cyber-pulse/config`

**权限**: 600

**格式**:
```bash
# Cyber Pulse API Configuration
# Generated by: api.sh configure

api_url=http://localhost:8000
admin_key=cp_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 4.3 命令清单

#### 配置命令

```bash
./scripts/api.sh configure              # 交互式配置 API URL 和 Admin Key
```

#### Sources 管理

```bash
# 已实现
./scripts/api.sh sources list [--status STATUS] [--tier TIER] [--scheduled BOOL]
./scripts/api.sh sources get <source_id>
./scripts/api.sh sources create --name NAME --connector-type TYPE --url URL [--tier TIER]
./scripts/api.sh sources update <source_id> [OPTIONS]
./scripts/api.sh sources delete <source_id>

# 待实现（另外任务完成）
./scripts/api.sh sources test <source_id>
./scripts/api.sh sources schedule <source_id> --interval SECONDS
./scripts/api.sh sources unschedule <source_id>
./scripts/api.sh sources import --file FILE
./scripts/api.sh sources export
./scripts/api.sh sources defaults
./scripts/api.sh sources set-defaults --interval SECONDS
```

#### Jobs 管理

```bash
./scripts/api.sh jobs list [--type TYPE] [--status STATUS] [--source-id ID]
./scripts/api.sh jobs get <job_id>
./scripts/api.sh jobs run <source_id>
```

#### Clients 管理

```bash
./scripts/api.sh clients list [--status STATUS]
./scripts/api.sh clients get <client_id>
./scripts/api.sh clients create --name NAME [--permissions PERMS] [--description DESC]
./scripts/api.sh clients rotate <client_id>
./scripts/api.sh clients suspend <client_id>
./scripts/api.sh clients activate <client_id>
./scripts/api.sh clients delete <client_id>
```

#### Logs 管理

```bash
./scripts/api.sh logs [--level LEVEL] [--source-id ID] [--since TIME]
```

#### Diagnose

```bash
./scripts/api.sh diagnose
```

## 5. 用户分离

### 5.1 用户类型

| 用户 | 获取方式 | 包含内容 |
|------|---------|---------|
| **开发者** | `git clone` | 完整代码库 |
| **运维人员** | 安装脚本 | 部署脚本 + 管理脚本 |

### 5.2 运维人员安装

```bash
# 安装脚本
curl -fsSL https://raw.githubusercontent.com/cyberstrat-forge/cyber-pulse/main/install.sh | bash
cd cyber-pulse
```

### 5.3 安装后目录结构（运维人员）

```
cyber-pulse/
├── scripts/
│   ├── cyber-pulse.sh      # 部署脚本
│   └── api.sh              # 管理脚本
├── deploy/
│   ├── docker-compose.yml
│   ├── docker-compose.prod.yml
│   ├── docker-compose.test.yml
│   └── init/
│       ├── check-deps.sh
│       └── generate-env.sh
├── sources.yaml            # 情报源配置
└── .env                    # 生成的配置（部署后）
```

**不含**: `src/`、`tests/`、`docs/`、`.git/`

### 5.4 开发者工作流

```bash
# 1. clone 代码库
git clone https://github.com/cyberstrat-forge/cyber-pulse.git
cd cyber-pulse

# 2. 开发环境（代码热重载）
./scripts/cyber-pulse.sh deploy --env dev --local

# 3. 测试环境（本地构建镜像）
./scripts/cyber-pulse.sh deploy --env test --local

# 4. 管理操作
./scripts/api.sh configure
./scripts/api.sh sources list
```

### 5.5 运维人员工作流

```bash
# 1. 安装
curl -fsSL https://raw.githubusercontent.com/cyberstrat-forge/cyber-pulse/main/install.sh | bash
cd cyber-pulse

# 2. 部署生产环境
./scripts/cyber-pulse.sh deploy --env prod

# 3. 配置管理脚本
./scripts/api.sh configure

# 4. 日常管理
./scripts/api.sh diagnose
./scripts/api.sh sources list
./scripts/api.sh jobs run src_xxx
```

### 5.6 用户对比

| 操作 | 开发者 | 运维人员 |
|------|--------|---------|
| 获取方式 | `git clone` | 安装脚本 |
| 目录内容 | 完整代码库 | 部署 + 管理脚本 |
| 部署命令 | `deploy --env dev/test --local` | `deploy --env prod` |
| 镜像来源 | 本地构建 | 远程拉取 |
| 管理脚本 | `./scripts/api.sh` | `./scripts/api.sh` |
