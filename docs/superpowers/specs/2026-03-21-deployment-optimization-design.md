## 1. 背景与目标

### 1.1 背景

cyber-pulse 当前已有基础的 Docker Compose 部署方案，但缺乏用户友好的部署和管理体验。用户需要：

- 简单快捷的一键部署
- 安全的配置管理
- 便捷的升级和回滚
- 独立的备份恢复能力

### 1.2 目标

让用户可以方便地从 GitHub 下载，然后一键式安装、运行和使用，也可以获得更新提醒，并安全地持续获得最新的应用功能。

### 4.1 命令概览

```bash
# 使用方式
./scripts/cyber-pulse.sh <command> [options]

# 命令列表
Commands:
  deploy [--env ENV]      一键部署（默认 dev 环境）
  start                   启动服务
  stop                    停止服务
  restart                 重启服务
  status                  查看状态（含健康检查）
  logs [SERVICE]          查看日志
  upgrade [--no-snapshot] 升级（默认自动快照保护）
  check-update            检查是否有新版本
  backup                  备份数据和配置
  restore <file>          从备份恢复
  uninstall               完全卸载
  help                    显示帮助
```

### 5.1 流程图

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤

│     ├─ Docker Compose 已安装                                │
│     ├─ 端口 8000/5432/6379 未被占用                         │
│     └─ 磁盘空间 ≥ 2GB                                       │
├─────────────────────────────────────────────────────────────┤
│  2. 配置初始化（如 .env 不存在）                             │
│     ├─ 生成随机密码（POSTGRES_PASSWORD）                     │
│     ├─ 生成随机 SECRET_KEY                                  │
│     └─ 写入 .env 文件，显示配置信息                          │
├─────────────────────────────────────────────────────────────┤
│  3. 版本检查                                                │
│     ├─ 查询 GitHub Releases 最新版本                        │
│     └─ 提示用户当前版本与最新版本                            │
├─────────────────────────────────────────────────────────────┤
│  4. 拉取/构建镜像                                           │
│     ├─ 优先使用预构建镜像（ghcr.io）                         │
│     └─ 无镜像时本地构建                                     │
├─────────────────────────────────────────────────────────────┤
│  5. 启动服务                                                │
│     ├─ docker-compose up -d                                 │
│     └─ 等待健康检查通过                                     │
├─────────────────────────────────────────────────────────────┤
│  6. 数据库迁移                                              │
│     └─ 执行 alembic upgrade head                            │
├─────────────────────────────────────────────────────────────┤
│  7. 部署完成                                                │
│     ├─ 显示访问地址                                         │
│     ├─ 显示配置信息位置 (.env)                              │
│     └─ 显示快速入门提示                                     │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 依赖检查细则

| 检查项 | 要求 | 失败提示 |
|--------|------|---------|
| Docker | ≥ 20.10 | "请安装 Docker 20.10 或更高版本" |
| Docker Compose | 已安装 | "请安装 Docker Compose" |
| 端口 8000 | 未占用 | "端口 8000 已被占用，请先释放或修改配置" |
| 端口 5432 | 未占用（容器内部） | 通过 Docker 网络隔离，无需检查 |
| 端口 6379 | 未占用（容器内部） | 通过 Docker 网络隔离，无需检查 |
| 磁盘空间 | ≥ 2GB | "磁盘空间不足，至少需要 2GB" |

### 5.3 安全配置生成

```bash
# 密码生成规则
POSTGRES_PASSWORD: 32 字符，包含大小写字母、数字、特殊字符
SECRET_KEY: 64 字符，随机 hex

# 配置文件示例
# .env（首次部署自动生成）
POSTGRES_USER=cyberpulse
POSTGRES_PASSWORD=<auto-generated>
SECRET_KEY=<auto-generated>
DATABASE_URL=postgresql://cyberpulse:<password>@postgres:5432/cyberpulse
REDIS_URL=redis://redis:6379/0
DRAMATIQ_BROKER_URL=redis://redis:6379/1
```

## 6. 升级流程设计

### 6.1 核心原则

- **快照和备份分离**：升级自动创建轻量快照，备份是独立的长期存储功能
- **自动回滚**：升级失败自动从快照恢复
- **用户无感知**：快照自动创建和管理，用户无需关心

### 6.2 快照 vs 备份

| 类型 | 目的 | 时机 | 持久性 | 内容 |
|------|------|------|--------|------|
| **快照** | 升级回滚 | 升级时自动创建 | 临时（成功后删除） | DB + 配置 |
| **备份** | 灾难恢复/迁移 | 用户手动或定时 | 长期保存 | 完整数据 + 配置 |

### 6.3 升级流程图

```
┌─────────────────────────────────────────────────────────────┐
│              cyber-pulse.sh upgrade                          │
├─────────────────────────────────────────────────────────────┤
│  1. 预检查                                                  │
│     ├─ 检查 Docker 可用                                     │
│     ├─ 检查磁盘空间（至少 1GB）                              │
│     └─ 检查当前服务健康                                     │
├─────────────────────────────────────────────────────────────┤
│  2. 检查更新                                                │
│     ├─ 查询 GitHub Releases                                 │
│     └─ 显示 Changelog 摘要                                  │
├─────────────────────────────────────────────────────────────┤
│  3. 创建快照（自动，~30s）                                   │
│     ├─ pg_dump → .upgrade-snapshot/database.dump            │
│     ├─ cp .env → .upgrade-snapshot/.env.backup              │
│     ├─ cp .version → .upgrade-snapshot/.version.backup      │
│     └─ 写入 metadata.json                                   │
├─────────────────────────────────────────────────────────────┤
│  4. 执行升级（~1-2min，服务短暂中断）                         │
│     ├─ git fetch --tags                                     │
│     ├─ git checkout <new-version>                           │
│     ├─ docker-compose build 或 docker pull                  │
│     ├─ alembic upgrade head                                │
│     └─ docker-compose up -d                                 │
├─────────────────────────────────────────────────────────────┤
│  5. 验证（~30s）                                            │
│     ├─ 等待服务就绪                                         │
│     ├─ 健康检查                                             │
│     ├─ 成功 → 删除快照，显示完成                             │
│     └─ 失败 → 执行回滚                                      │
├─────────────────────────────────────────────────────────────┤
│  6. 回滚（如失败，~1min）                                    │
│     ├─ 从快照恢复数据库                                     │
│     ├─ 恢复 .env 和 .version                                │
│     ├─ git checkout 旧版本                                  │
│     ├─ 重启服务                                             │
│     └─ 保留快照供排查                                       │
└─────────────────────────────────────────────────────────────┘
```

### 6.4 升级时间估算

| 步骤 | 使用预构建镜像 | 本地构建 |
|------|---------------|---------|
| 预检查 | ~3s | ~3s |
| 检查更新 | ~5s | ~5s |
| 创建快照 | ~30s | ~30s |
| 代码切换 | ~5s | ~5s |
| 镜像拉取/构建 | ~30s | ~2-5min |
| DB 迁移 | ~5-30s | ~5-30s |
| 服务重启 | ~10s | ~10s |
| 验证 | ~30s | ~30s |
| **总计** | **~2min** | **~4-7min** |

### 6.5 服务中断说明

```
升级时间线：
  ┌─────────────────────────────────────────────────────┐
  │ 服务正常运行                                         │
  ├─────────────────────────────────────────────────────┤
  │ [快照] [代码切换] [构建] [DB迁移] [重启] [验证]      │
  │                                              ↓       │
  │                                    服务中断 ~1-2min  │
  └─────────────────────────────────────────────────────┘
```

## 7. 备份恢复设计

### 7.1 备份内容

| 类型 | 内容 | 说明 |
|------|------|------|
| 数据库 | PostgreSQL 完整导出 | 所有业务数据 |
| 配置文件 | .env | 密码、密钥等敏感配置 |
| 版本信息 | .version | 当前运行版本 |
| 应用数据 | data/ 目录 | 用户数据（如有） |

### 7.2 备份流程

```
┌─────────────────────────────────────────────────────────────┐
│                    cyber-pulse.sh backup                     │
├─────────────────────────────────────────────────────────────┤
│  1. 创建备份目录                                            │
│     backups/backup-YYYYMMDD-HHMMSS/                         │
├─────────────────────────────────────────────────────────────┤
│  2. 备份数据库                                              │
│     docker exec postgres pg_dump -U cyberpulse cyberpulse   │
│     > database.sql                                          │
├─────────────────────────────────────────────────────────────┤
│  3. 备份配置文件                                            │
│     .env → backup.env                                       │
│     .version → backup.version（如存在）                      │
├─────────────────────────────────────────────────────────────┤
│  4. 备份应用数据（如存在）                                    │
│     data/ → data/（非空时）                                  │
├─────────────────────────────────────────────────────────────┤
│  5. 生成备份元数据                                          │
│     backup.json:                                            │
│       - backup_time: 时间戳                                  │
│       - version: 当前版本                                    │
│       - db_size: 数据库大小                                  │
│       - checksum: 校验和                                    │
├─────────────────────────────────────────────────────────────┤
│  6. 打包压缩                                                │
│     tar -czf backup-YYYYMMDD-HHMMSS.tar.gz                  │
├─────────────────────────────────────────────────────────────┤
│  7. 清理旧备份（保留最近 5 个）                               │
└─────────────────────────────────────────────────────────────┘
```

### 7.3 恢复流程

```
┌─────────────────────────────────────────────────────────────┐
│                    cyber-pulse.sh restore                    │
├─────────────────────────────────────────────────────────────┤
│  1. 解压备份文件                                            │
├─────────────────────────────────────────────────────────────┤
│  2. 恢复配置文件                                            │
│     backup.env → .env                                       │
│     backup.version → .version                               │
├─────────────────────────────────────────────────────────────┤
│  3. 恢复数据库                                              │
│     ├─ 停止 API/Worker/Scheduler（避免写入）                │
│     ├─ psql < database.sql                                  │
│     └─ 重启服务                                             │
├─────────────────────────────────────────────────────────────┤
│  4. 恢复应用数据                                            │
│     data/ → data/                                           │
├─────────────────────────────────────────────────────────────┤
│  5. 切换代码版本                                            │
│     git checkout $(cat .version)                            │
├─────────────────────────────────────────────────────────────┤
│  6. 重建并启动服务                                          │
└─────────────────────────────────────────────────────────────┘
```

## 8. 多环境支持

### 8.1 环境配置文件

```yaml
# docker-compose.yml（基础配置）
services:
  postgres:
    image: postgres:15
    # 共用配置...

# docker-compose.prod.yml（生产覆盖）
services:
  postgres:
    volumes:
      - postgres_prod_data:/var/lib/postgresql/data
    deploy:
      resources:
        limits:
          memory: 2G
  api:
    command: uvicorn cyberpulse.api.main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 8.2 使用方式

```bash
# 开发环境（默认）
./scripts/cyber-pulse.sh deploy

# 测试环境
./scripts/cyber-pulse.sh deploy --env test

# 生产环境
./scripts/cyber-pulse.sh deploy --env prod
```

## 11. 安全考虑

### 11.1 密码管理

- 首次部署自动生成强密码
- 密码存储在 `.env` 文件中
- 用户可通过 `./cyber-pulse.sh config` 查看配置

### 11.2 文件权限

```bash
# .env 文件权限
chmod 600 .env
```

### 11.3 网络隔离

- PostgreSQL 和 Redis 不对外暴露端口
- 仅 API 服务暴露 8000 端口
